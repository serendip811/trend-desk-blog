# Lessons

- 정적 페이지를 여러 HTML 파일에서 직접 중복 관리하면 홈, 아카이브, 카테고리, RSS, 사이트맵이 쉽게 어긋난다. 글 메타데이터는 반드시 단일 원본에서 관리한다.
- 목록 UI와 본문 UI의 요구사항을 분리한다. 글 본문에 복원한 이미지를 목록 썸네일까지 자동 노출하지 않는다.
- 공용 피드 레이아웃을 유지할 때는 홈과 아카이브, 카테고리의 마크업 구조를 맞춰야 한다. 같은 클래스 이름이라도 DOM 구조가 다르면 레이아웃이 깨진다.
- 중복 주제 글을 즉시 삭제하기보다 `listed`, `rss`, `sitemap` 같은 공개 여부 플래그로 먼저 통제하면 기존 링크를 덜 깨뜨리고 정리하기 쉽다.
